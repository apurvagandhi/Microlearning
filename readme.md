- **Text-based learning** - concise lessons and summaries
- **Visual learning** - diagrams, infographics, and illustrations
- **Audio learning** - podcasts, spoken explanations
- **Writing practice** - exercises and quizzes

## üóìÔ∏è Month-long Hackathon (10 Hands-on Days)

The project runs over approximately one month. During this period there are 10 dedicated "hands-on" coding days where students will implement, test, and integrate features. The remaining days are reserved for planning, design, setup, reviews, and presentations.

### Week 1: Planning & Architecture
- [ ] Introduction to project
- [ ] Set up project management (GitHub Discussions, Projects board)
- [ ] Set up Google Chat
- [ ] Set up GitHub repository and project structure 

### Week 2: Design & Setup
- [ ] Discuss, Decide and Design complete architecture
- [ ] Discuss, Decide and Define API endpoints
- [ ] Discuss, Decide and Create database schema
- [ ] Discuss, Decide and Design UI mockups
- [ ] Set up local environment
- [ ] Discuss, Decide and Set up CI/CD and AWS resources

### Weeks 3 & 4: Implementation ‚Äî 10 Hands-on Days
- [ ] Frontend: Layout, authentication UI
- [ ] Backend: Authentication, user management, database setup
- [ ] Integration: Frontend ‚Üî Backend
- [ ] Iterative feature development, code reviews, and testing

### Final Days: Polish, Testing & Presentation
- [ ] Performance optimization
- [ ] Cross-browser testing
- [ ] Documentation
- [ ] Deployment to production
- [ ] Prepare presentation and live demo

Notes:
- The schedule is flexible; organizers may distribute the 10 hands-on days across Weeks 3 & 4 as needed.
- Use GitHub Issues and Projects to mark which days are hands-on and to assign responsibilities.

## üìû Communication & Project Management

### Primary Channels
1. **GitHub Discussions** - Technical decisions, architecture debates
2. **GitHub Issues** - Bug reports, feature requests, tasks
3. **GitHub Projects** - Sprint tracking, task assignments
4. **Pull Requests** - Code reviews, knowledge sharing
5. **Group Chat** - Google Chat for Real-time chat, quick questions

### Workflow
1. Create an issue for every task
2. Assign to yourself or discuss in the issue
3. Create a branch: `git checkout -b feature/issue-number-name`
4. Submit PR with issue reference: `Closes #123`
5. Request 1 review before merge
6. Merge when approved
